# Applied Econometrics Projects

Repository built to complete exercises and projects from the Applied Econometrics course.

---

## Project description

Weekly applied exercises using two datasets:

- **Airbnb listings** (weeks 1–2): price analysis, OLS regression, AI-generated neighbourhood scores
- **US state-year fatality panel** (week 3+): panel data methods

---

## Repository structure

```
Applied_Econometrics_Projects/
├── datasets/
│   ├── airbnb/
│   │   ├── _config_airbnb.R          ← set REGION here
│   │   ├── listings/                  ← raw CSVs (10 Italian cities)
│   │   ├── neigh_ratings/             ← manually collected Milan ratings
│   │   ├── neigh_scores/              ← AI-generated neighbourhood scores (Puglia)
│   │   └── processed/                 ← generated by scripts (gitignored)
│   └── US_state_year/
│       ├── _config_fatality.R         ← set YEAR_RANGE here
│       ├── fatality.dta               ← raw panel data
│       └── processed/                 ← generated by scripts (gitignored)
├── week_1/
│   ├── build/code/clean_data_task1.R  ← cleans raw listings
│   ├── analysis/code/price_analysis.R ← summary stats, plots, OLS (tasks 2–5)
│   └── report/Week_1.qmd
├── week_2/
│   ├── build/code/data_prep.R         ← standardises rating variable
│   ├── analysis/
│   │   ├── code/data_analysis.R       ← OVB, AI scores, t-test, CI, F-test
│   │   └── input/agents/              ← AI methodology docs (prompt + process notes)
│   └── report/Week_2.qmd
├── week_3/
│   ├── build/code/data_setup.R        ← loads fatality panel
│   ├── analysis/code/
│   └── report/
├── tasks/                             ← weekly task PDFs
├── run_all.R                          ← runs full pipeline end-to-end
└── PLAN.md                            ← restructuring plan (session reference)
```

## Data flow

```
datasets/airbnb/listings/{REGION}.csv
  → week_1/build/code/clean_data_task1.R
      → datasets/airbnb/processed/listings_{region}_cleaned.rds
          → week_2/build/code/data_prep.R
              → datasets/airbnb/processed/listings_{region}_week2.rds
                  → week_2/analysis/code/data_analysis.R  (+ neigh_scores/)
                      → datasets/airbnb/processed/listings_{region}_final.rds

datasets/US_state_year/fatality.dta
  → week_3/build/code/data_setup.R
      → datasets/US_state_year/processed/fatality_panel.rds
```

All processed data lives in `datasets/*/processed/` and is gitignored (reproducible from scripts).

---

## How to change region / parameters

**Airbnb analysis (weeks 1–2):** edit `REGION` in `datasets/airbnb/_config_airbnb.R`

```r
REGION <- "Milan"   # any city in datasets/airbnb/listings/
```

**Fatality analysis (week 3+):** edit `YEAR_RANGE` in `datasets/US_state_year/_config_fatality.R`

```r
YEAR_RANGE <- c(1982, 1988)
```

## AI-generated neighbourhood scores

For week 2, neighbourhood-level scores (Coolness, Centrality, Quietness, Fanciness) were
generated manually using three AI agents with identical prompts:

| Agent | Weight | File |
|-------|--------|------|
| ChatGPT | 0.25 | `datasets/airbnb/neigh_scores/puglia_scores_chatgpt.csv` |
| Gemini (Thinking) | 0.25 | `datasets/airbnb/neigh_scores/puglia_scores_gemini.csv` |
| Perplexity Deep Research | 0.50 | `datasets/airbnb/neigh_scores/puglia_scores_perplexity.csv` |

Process documentation (prompt, agent logs): `week_2/analysis/input/agents/`
